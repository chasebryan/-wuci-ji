Profiles

Separated by design.

Noether Core

Invariant-preserving base profile. TTY-first, GUI-free, network-minimized, evidence-bound, and authoritative for the serious base.

Birkhoff Bastion

Optional operator shell profile. Ratpoison and DWM are candidates only and must not enter Noether Core.

Tarski Review Appliance

Reviewer evidence generator. It produces review packets, surface reports, substrate matrices, score material, and non-claim boundaries.

Developer Desktop

Non-authoritative convenience profile. Xfce and browsers belong here only if retained at all.

Euclid Substrate Trial

Substrate selection requires measured trial evidence.

Buildroot, Alpine, Debian minimal, Void, NixOS, Guix, Yocto, and OpenBSD reference are candidates. Void remains a candidate only. Missing trial metrics are recorded as NOT_MEASURED.

Phase 1 prepares the first artifact cohort: Buildroot, Alpine, and Debian minimal. It does not rank or select a substrate.

Phase 2B expands safe build feasibility probes to the full original substrate candidate set: Buildroot, Alpine, Debian minimal, Void, NixOS, Guix, Yocto, and OpenBSD reference. Attempt mode is not part of default validation.

Phase 3A defines controlled build rooms for the full cohort. The build room is the measuring chamber, not the substrate, and it does not execute builds.

Phase 3B readiness inspects backend, input, policy, and resource blockers for those build rooms. It authorizes only L0 readiness checks by default and does not execute build rooms.

Phase 3C-A verifies rootless backend smoke mechanics with a synthetic non-substrate image. The smoke image is not a WuciOS artifact, not a substrate artifact, and not score eligible.

Phase 3C-B defines direct-rootfs preparation policy for Buildroot, Alpine, Debian minimal, and Void. It does not generate rootfs images, attempt substrate artifacts, select a substrate, rank candidates, or generate a numeric WuciOS score.

Phase 3C-C defines NixOS/Guix store-root preparation policy. It does not build or run NixOS or Guix, realize store paths, generate rootfs images, attempt substrate artifacts, select a substrate, rank candidates, generate artifact hashes, or generate a numeric WuciOS score.

Phase 3C-D defines Yocto layer/recipe preparation policy. It does not run BitBake, initialize a Yocto build environment, clone or download Yocto sources or layers, generate rootfs or image outputs, attempt substrate artifacts, select a substrate, rank candidates, generate artifact hashes, or generate a numeric WuciOS score.

Phase 3C-E defines OpenBSD reference preparation policy. It does not install or boot OpenBSD, inspect runtime behavior, run package/admin commands, clone source trees, download ports trees or install media, launch VMs, generate rootfs or image outputs, attempt substrate artifacts, select a substrate, rank candidates, generate artifact hashes, or generate a numeric WuciOS score.

Phase 3C is integrated and closed at origin/wucios-v24-reduction-gate commit 0f06b62. No WuciOS artifact, artifact hash, substrate selection, ranking, or numeric WuciOS score exists from Phase 3C. L3 and L4 remain unauthorized. No next implementation phase is authorized or inferred by this closeout.

The later WuciOS v2.4 Alpine Substrate Trial produced a bounded trial artifact under build/wucios/full-trial/alpine/ from an Alpine release minirootfs 3.24.1 x86_64 input recorded in generated input evidence. The artifact SHA-256 is 95f3cb750357eeb2cd840ddfc7b62c6addf08b2d66079871a4d8e8efdd9ae45f. Generated score evidence file build/wucios/full-trial/alpine/wucios-alpine-substrate-trial-score.json records a WuciOS v2.4 Alpine Substrate Trial Score of 96.0 / 100.0. External validation is not claimed, production readiness is not claimed, and full runtime validation is not claimed. Runtime Gates 5 through 9 produced local/ignored runtime evidence only; raw runtime evidence remains uncommitted unless separately authorized.

Gödel Boundary

What WuciOS v2.4 does not claim.

WuciOS v2.4 is not externally certified, not production authorized, not government approved, not seL4 equivalent, not a formally verified kernel claim, not perfect security, and not a claim that it cannot be hacked. Reduced size alone does not prove security.

This Daylight site status is not a WuciOS substrate score and is not a WuciOS release-authoritative artifact score. Diagnostic score material remains INVALID_WITHOUT_ARTIFACT until artifact-bound evidence exists.
