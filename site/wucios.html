Profiles

Separated by design.

Noether Core

Invariant-preserving base profile. TTY-first, GUI-free, network-minimized, evidence-bound, and authoritative for the serious base.

Birkhoff Bastion

Optional operator shell profile. Ratpoison and DWM are candidates only and must not enter Noether Core.

Tarski Review Appliance

Reviewer evidence generator. It produces review packets, surface reports, substrate matrices, score material, and non-claim boundaries.

Developer Desktop

Non-authoritative convenience profile. Xfce and browsers belong here only if retained at all.

Euclid Substrate Trial

No substrate is selected without measurements.

Buildroot, Alpine, Debian minimal, Void, NixOS, Guix, Yocto, and OpenBSD reference are candidates. Void remains a candidate only. Missing trial metrics are recorded as NOT_MEASURED.

Phase 1 prepares the first artifact cohort: Buildroot, Alpine, and Debian minimal. It does not rank or select a substrate.

Phase 2B expands safe build feasibility probes to the full original substrate candidate set: Buildroot, Alpine, Debian minimal, Void, NixOS, Guix, Yocto, and OpenBSD reference. Attempt mode is not part of default validation.

Phase 3A defines controlled build rooms for the full cohort. The build room is the measuring chamber, not the substrate, and it does not execute builds.

Phase 3B readiness inspects backend, input, policy, and resource blockers for those build rooms. It authorizes only L0 readiness checks by default and does not execute build rooms.

Gödel Boundary

What WuciOS v2.4 does not claim.

WuciOS v2.4 is not externally certified, not production authorized, not government approved, not seL4 equivalent, not a formally verified kernel claim, not perfect security, and not a claim that it cannot be hacked. Reduced size alone does not prove security.

This Daylight site status is not a WuciOS substrate score and is not a WuciOS release-authoritative artifact score. Diagnostic score material remains INVALID_WITHOUT_ARTIFACT until artifact-bound evidence exists.
