Enterprise Adoption

Use Daylight as an evidence layer without replacing existing security products.

Local Mode

Run schema, example, and claim-boundary checks before publishing claims.

CI Mode

Run the copyable GitHub Action or local Make targets on pull requests.

Release-Gate Mode

Block release, publish, install, trust, decrypt, or deploy when evidence is missing.

Compliance-Map Mode

Export evidence-to-control maps as review aids, not certifications.

Monitoring Mode

Accept monitor signals that can downgrade or revoke claim authority.

Security-Product Evaluation Mode

Score product-standard readiness separately from security-performance claims.

Do Not Use It For Yet

Endpoint protection, malware detection, network firewalling, general runtime sandboxing, production cryptography, certification claims, government authorization, or automatic vulnerability fixing.
