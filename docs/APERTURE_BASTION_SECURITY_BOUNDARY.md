# Aperture Bastion Security Boundary

This file is blunt on purpose. A boundary is listed as enforced only when
`daylight/v19-aperture-bastion/` has code and tests for that exact behavior
at the current commit. Every row also states what the boundary does **not**
prove, because that is where overclaims start.

| Boundary | What it checks | Owner | Current status | What it does not prove |
| --- | --- | --- | --- | --- |
| Capsule digest | SHA-256 over domain separator + canonical JSON (sorted keys, stable separators, no floats) of every capsule field except the digest; recomputed on every load | `src/capsule.py` (`capsule_digest`, `validate_capsule_shape`); `tests/test_capsule.py` | Enforced | That capsule contents are true; a digest binds bytes, it does not certify facts. Anyone can re-digest an edited capsule — which is why claim checks are also semantic. |
| Subject digest | Streamed SHA-256 + SHA3-512 + size for every input subject; re-hashed against disk at verify | `src/capsule.py`, `src/pathsafe.py`; `tests/test_capsule.py` | Enforced | That the subject is safe, correct, or built reproducibly. It proves the reviewed bytes are these bytes, nothing more. |
| Public manifest | Sorted unique normalized repo-relative paths; no absolute paths, no `..`, no backslash/drive/NUL; per-file SHA-256 + size; `public_sha256sums` recomputed; reserved names refused | `src/capsule.py` (`_validate_manifest`), `src/pathsafe.py`; `tests/test_capsule.py`, `tests/test_public_artifact.py` | Enforced | That listed files are appropriate to publish beyond the firewall rules; a clean manifest of the wrong file is still the wrong file. |
| Public artifact firewall | Recursive scan: private names/suffixes/patterns, private directories, content markers, key-shaped hex, plaintext-oracle JSON, symlinks, hardlinks, hidden components, oversize files, unexpected files, missing manifest files, SHA256SUMS drift; report written only on pass, only outside the root | `src/firewall.py`, `src/profile.py`; `tests/test_firewall.py` | Enforced | Absence of all secrets. It matches a pinned finite rule profile; a secret that matches no rule (novel format, encrypted blob, steganography) passes. It is a tripwire, not a classifier. |
| Meridian scorecard reference | Exact file bytes pinned by SHA-256; `manual_edit_allowed`/`manual_override` must be false; `final_score_M` must equal term-contribution sum; perfect scores require zero open obligations plus non-self-signed external attestations; perfect references must be re-checkable at verify | `src/evidence_refs.py`; `tests/test_evidence_refs.py` | Enforced (consistency gates) | Full Meridian correctness. The q-vector re-derivation lives in the Meridian verifier (`make daylight-meridian-verify`), not here. These gates catch edits and unsupported perfect claims, not subtle upstream scoring bugs. |
| Binaric vector reference | `vector_digest` recomputed with the v18 domain string; `previous_vector_digest` chain link required and matched for every non-first vector; head pinned into the capsule | `src/evidence_refs.py`; `tests/test_evidence_refs.py` | Enforced | Host cleanliness, runtime state, or that the measured binary is benign — exactly the v18 non-claims, inherited unchanged. |
| Transition ledger reference | Genesis head recomputed; per-entry `previous_head` linkage and `head = H(previous_head, entry_digest)` recomputed; final head pinned | `src/evidence_refs.py`; `tests/test_evidence_refs.py` | Enforced | Transition legitimacy. Ledger chain integrity is not user-ceremony verification; that stays in v18 (`tamper-check`). |
| Public artifact | Built only from a fully verified capsule; sources re-hashed and re-scanned at copy time; atomic writes; non-empty output requires `--force`; failed builds remove their own partial output; deterministic SHA256SUMS | `src/public_artifact.py`; `tests/test_public_artifact.py` | Enforced | That publication is authorized or wise. It proves the directory matches the capsule, not that the capsule should be published. |
| Private-material exclusion | Same profile applied three times: capsule build (pre-publication), artifact copy, and post-build firewall; profile id + digest pinned inside every capsule | `src/profile.py` + call sites; `tests/test_capsule.py`, `tests/test_firewall.py` | Enforced | Exhaustive secrecy protection. The profile is a deny-list plus structural rules; it reduces accident risk, it does not make leaks impossible. |
| External attestation handling | Attestations count as external only with a `signer_id` not matching self patterns (`self`, `internal`, `local`, `repo`, `harness`, `wuci-ji`) and no `self_signed: true`; self-signed closure behind a perfect score rejects | `src/evidence_refs.py`; `tests/test_evidence_refs.py` | Enforced (negative gate only) | Attestation authenticity. No signature is cryptographically verified here — signed external evidence verification is Solstice's lane. This gate can only refuse obvious self-closure; it cannot validate a genuine external party. |
| CI upload safety | Workflow has `permissions: contents: read`; upload path is the firewalled public dir only; `if-no-files-found: error`; no `if: always()`; firewall steps precede upload; checked by the repo-wide `check-workflow` lane | `.github/workflows/daylight-v19-aperture-bastion.yml`; `tools/daylight_public_evidence_firewall.py` | Enforced | GitHub-side behavior beyond the workflow text, or that uploaded artifacts are consumed safely downstream. |
| Claim wording | `claim_boundary` keys are a closed set that must all be exactly `false`; mandatory non-claims must be present, sorted, unique; violations reject even when the capsule digest is recomputed | `src/claims.py`, `src/capsule.py`; `tests/test_claims.py`, `tests/test_capsule.py` | Enforced | That prose elsewhere is equally careful. The capsule cannot overclaim; humans writing READMEs still can. |

## Standing non-claims for the whole layer

Aperture Bastion is not production cryptography, not runtime containment or
sandboxing, not host cleanliness proof, not FIPS validated, not government
validated, without external-certification evidence from any outside auditor, not whole-system post-quantum safe, not
an independent audit, and not a perfect Daylight score claim from
repository-owned evidence. Its verification chain runs on an untrusted host
with repository-owned fixtures; every guarantee above is exactly as strong
as that trust base and no stronger.
