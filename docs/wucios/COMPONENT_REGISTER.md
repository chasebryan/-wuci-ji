# Component Register

The component register is [wucios/components/component-register.json](../../wucios/components/component-register.json).

Every included component needs:

- a reason
- profile membership
- component class
- network surface statement
- privilege surface statement
- language surface
- dependency notes
- risk notes
- acceptance gate
- evidence hooks
- rejection conditions

Components without a register entry do not enter Noether Core. Ratpoison and DWM are candidate-only in Birkhoff Bastion. Xfce is demoted to Developer Desktop and must not be used as a high-assurance base profile.
