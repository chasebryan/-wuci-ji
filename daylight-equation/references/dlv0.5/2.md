0. V0.5 DELTA FROM V0.4

v0.5 changes the core in five ways:

1. review_receipt_hash is removed from the pre-encryption header.
   Reason: v0.4 created a circular dependency:
     header -> T0 -> K_COM -> com_A -> review_receipt -> review_receipt_hash -> header.

2. aead_id is moved into the header.
   Reason: the selected AEAD is now bound by T0, AEAD associated data, KEM schedule, and authorization.

3. The encrypted plaintext is now a typed PrivatePayload, not raw artifact bytes.
   Reason: reviewed_content needs a hidden review opening without leaking H(A).

4. policy_hash, keyset_hash, claims_hash, provenance_hash, and manifest_hash are hashes of actual supplied or resolved objects.
   Reason: a hash alone is not parseable and cannot satisfy PolicyOK.

5. ML-KEM malformed-ciphertext handling is clarified.
   Reason: a bad ML-KEM ciphertext may fail by AEAD/commit failure, not necessarily by explicit Decaps rejection.


1. DESIGN RULE

Daylight defines no new cryptographic primitive.

Daylight_Core =
  TypedEnvelope
  + DeterministicEncoding
  + HybridKEM(ML-KEM-1024, DHKEM-P384-HKDF-SHA384)
  + AEAD
  + Auth_Q
  + PolicyGate
  + FailClosedOpen

Daylight_Root =
  Daylight_Core
  + Auth_H
  + NoDowngradeFinal

Daylight_Audit =
  Daylight_Root
  + TransparencyLogProofs

Daylight_FROST_Extension =
  Daylight_Audit
  + Auth_F

Install safety, witness evidence, audit logs, and FROST are not cryptographic confidentiality primitives.

Custom components are forbidden in the core unless they are:
  typed,
  canonically encoded,
  byte-level specified,
  test-vectored,
  fuzzed,
  formally modeled where applicable,
  and externally reviewed.

3. CANONICAL ENCODING

ENC = Deterministic-CBOR-Daylight-v5

Allowed CBOR major types:
  unsigned integer,
  byte string,
  text string,
  array,
  map,
  true,
  false,
  null.

Forbidden in core encoding:
  negative integers,
  floating point,
  CBOR tags,
  indefinite-length items,
  duplicate map keys,
  unknown map keys,
  unknown critical fields,
  non-minimal integers,
  non-minimal lengths,
  unordered maps,
  unordered rosters,
  simple values other than true, false, null.

All schema maps use unsigned-integer keys only.

Map ordering:
  keys MUST be strictly increasing by numeric value.

Roster ordering:
  roster entries MUST be sorted by key_id ascending byte lexicographic order.

Array ordering:
  arrays are ordered exactly as specified by their type.

DECODE_AS(T,b) = bottom if:
  b is not deterministic CBOR,
  b does not match type T,
  a required field is absent,
  an unknown field is present,
  a duplicate field is present,
  a field has the wrong type,
  a roster is not sorted,
  or an integer is outside its allowed domain.

ENC(x) = ENC(y) => x = y
DECODE_AS(T,ENC(x)) = x if x has type T

E(label,x1,...,xn) = ENC([label,x1,...,xn])

Labels are ASCII text strings and MUST match exactly.

10. PUBLIC LEAKAGE AND REVIEW COMMITMENTS

For artifact A:

artifact_len  = len(A)
artifact_hash = ArtifactHash(A)

If content_scope = metadata_only:
  leak_value = artifact_len
  review_blind = null

If content_scope = public_commitment:
  leak_value = [artifact_len, artifact_hash]
  review_blind = null

  sample rho_R in {0,1}^256
  review_commit =
    Hb(ENC([
      "daylight.review.hidden.v5",
      rho_R,
      artifact_len,
      artifact_hash,
      policy_hash,

12. HYBRID KEM BLOCK

KEMBlock_v5 = map {
  0: q_kem_key_id,
  1: c_kem_key_id,
  2: enc_Q,
  3: enc_C
}

q_kem_key_id : Hash64

KEMBlockOK =
  1[q_kem_key_id identifies ek_Q in keyset_obj]
  * 1[c_kem_key_id identifies pk_C in keyset_obj]
  * MLKEMCiphertextParseOK(enc_Q)
  * DHKEMEncapsulationParseOK(enc_C)
  * PublicKeyValidate(ek_Q)
  * PublicKeyValidate(pk_C)

Seal side:

(ss_Q, enc_Q) = ML-KEM-1024.Encaps(ek_Q)
(ss_C, enc_C) = DHKEM-P384-HKDF-SHA384.Encap(pk_C)

kem_context =
  E(
    "daylight.kem.context.v5",
    suite_id,

    H(T0),

    q_kem_key_id,
    c_kem_key_id,

    "daylight.kem.salt.v5",

    H(T0),
    keyset_hash,
    q_kem_key_id,
    c_kem_key_id
  ])

PRK_D =
  KDF.Extract(
    salt_kem,
    ENC([
      "daylight.hybrid.ikm.v5",
      ss_Q,
      ss_C,
      kem_context
    ])
  )

OKM =
  KDF.Expand(
    PRK_D,
    ENC([
      "daylight.key.schedule.v5",
      H(T0),
      H(kem_block),
      suite_id,
      profile,

      aead_id
    ]),
    76
  )

Split(OKM) =
  K_E    = OKM[0..31]
  K_COM  = OKM[32..63]
  N_base = OKM[64..75]

Open side:

ss_Q = ML-KEM-1024.Decaps(dk_Q, enc_Q)
ss_C = DHKEM-P384-HKDF-SHA384.Decap(sk_C, enc_C)

If KEMBlockOK = 0:
  Open(omega) = bottom

If DHKEM decapsulation rejects:
  Open(omega) = bottom

If ML-KEM ciphertext has invalid encoding or wrong length:
  Open(omega) = bottom

If ML-KEM implicit-rejection decapsulation returns a secret for a modified but parse-valid ciphertext:
  failure is expected to occur at AEAD.Dec or CommitOK.


13. HYBRID KEM SECURITY TARGET

HybridSecretOK =
  ss_Q is pseudorandom
  OR
  ss_C is pseudorandom

HybridKEM claim:

K_E, K_COM, and N_base are pseudorandom if:
  HybridSecretOK = 1,
  HKDF-SHA512 behaves as a PRF/extractor over the encoded input domain,
  public keys are validated,
  ENC is injective,
  KEMBlockOK = 1,
  and implementation side channels are excluded or bounded.

Bound:

Adv_hybrid_KEM <=
  min(
    Adv_IND_CCA_ML_KEM_1024,
    Adv_SS_DHKEM_P384
  )
  + Adv_PRF_HKDF_SHA512
  + eps_key_validation
  + eps_encoding
  + eps_binding
  + eps_impl


14. NONCE RULE

For record index j:

N_j = N_base XOR I2OSP_96(j)

NonceOK(j) = 1[0 <= j < 2^96]

Core profile:
  single-shot only
  j = 0
  N0 = N_base

Global nonce invariant:

For all envelopes e,e2 and record indices j,j2:

  (K_E(e), N_j(e)) = (K_E(e2), N_j2(e2))
  =>
  (e,j) = (e2,j2)

Nonce reuse is a critical break.

23. NO-DOWNGRADE

  (r_T, mu_T, suite_id_T, profile_T, aead_id_T)

  1[r_T >= r_min]
  * 1[ord(mu_T) >= ord(mu_min)]
  * 1[suite_id_T = suite_id]
  * 1[aead_id_T in AllowedAEAD(policy_obj)]

  1[r_T >= r_prev]
  * 1[ord(mu_T) >= ord(mu_prev)]

  1[action = root_rotate]
  * V_Q
  * V_H
  * PolicyAllowsRootRotate(policy_obj,T0)

  NoDowngrade(T0)
  * (
      LedgerModeMonotone(T0)
      OR
      RootRotateException(T0)

    * ReproBuildOK(build_hash)
    * AttestationOK(attestation_hash)

Allowed meaning:
  WitnessOK = NoPrivateMaterialFoundUnderDeclaredScope

  WitnessOK = NoPrivateMaterialExists


27. PUBLIC PRECHECK

PublicPreOK(omega) =
  ParseOK(omega)
  * SuiteOK(omega)
  * AuxHashOK
  * KEMBlockOK
  * ModeOK(profile,r,mu,action)
  * PolicyOK
  * ClaimOK(claims_obj,r)
  * GateOK
  * ProvenanceOK
  * ContentReviewPreOK
  * V_Auth(profile,r,mu)
  * NoDowngradeFinal(T0)
  * LogOK
  * InstallOK
  * WitnessOK

Important ordering rule:

If PublicPreOK(omega) = 0:
  Open(omega) = bottom
  private KEM decapsulation MUST NOT be called
  AEAD.Dec MUST NOT be called
  plaintext MUST NOT be materialized


28. PRIVATE OPEN CHECK

DeriveOK =
  KEMBlockOK
  * DecapOK_Q
  * DecapOK_C
  * NonceOK(0)

P_prime = AEAD.Dec(K_E, N0, C, AD = T0)

DecOK = 1[P_prime != bottom]

PayloadOK(P_prime) =
  1[DECODE_AS(PrivatePayload_v5, P_prime) != bottom]

A_prime = P_prime.artifact
review_blind_prime = P_prime.review_blind

CommitOK(A_prime) =
  1[
    KDF2(
      K_COM,
      "daylight.artifact.commit.v5",
      (
        H(T0),
        Hb(C),
        len(A_prime),
        ArtifactHash(A_prime),
        leak_value
      ),
      32
    )
    =
    com_A
  ]

LeakOK(A_prime, review_blind_prime, leak_value) =
  ContentReviewPrivateOK(A_prime, review_blind_prime, header)

PrivateOpenOK(A_prime,omega) =
  DeriveOK
  * DecOK
  * PayloadOK(P_prime)
  * CommitOK(A_prime)
  * LeakOK(A_prime, review_blind_prime, leak_value)


29. MAIN OPEN ALGORITHM

Open(omega):

  if PublicPreOK(omega) = 0:
    return bottom

  derive ss_Q, ss_C, K_E, K_COM, N0

  if DeriveOK = 0:
    return bottom

  P_prime = AEAD.Dec(K_E, N0, C, AD = T0)

  if P_prime = bottom:
    return bottom

  if PayloadOK(P_prime) = 0:
    zeroize P_prime
    return bottom

  A_prime = P_prime.artifact

  if PrivateOpenOK(A_prime,omega) = 0:
    zeroize A_prime
    zeroize P_prime
    return bottom

  zeroize review_blind_prime
  return A_prime


30. MAIN ACCEPTANCE PREDICATE

DL(profile,r,mu,A_prime,omega) =
  PublicPreOK(omega)
  * PrivateOpenOK(A_prime,omega)

Open(omega) != bottom
  iff
exists A_prime such that:
  DL(profile,r,mu,A_prime,omega) = 1

38. QUORUM MODEL

P_avail(p,n,t) =
  sum from i=t to n of C(n,i) * p^i * (1-p)^(n-i)

P_break(c,n,t) =
  sum from i=t to n of C(n,i) * c^i * (1-c)^(n-i)

P_quorum_break <=
  P_break(c,n,t)
  + P_domain_break(kappa,D,u)
  + p_corr

IndependenceAssumptionAllowed = 1 only if p_corr is explicitly modeled.

If domain_count is undefined:
  V_Q = 0
  V_F = 0
